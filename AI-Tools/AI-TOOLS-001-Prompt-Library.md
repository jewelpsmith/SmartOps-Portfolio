## 💡 Why This Library Exists

A bad prompt gets you a generic answer. A good prompt 
gets you something you can actually use.

Most people treat AI tools like a search engine; they 
type a vague question and hope for the best. The people 
getting real value out of these tools understand that 
the quality of your output is directly tied to the 
quality of your input. Prompting is a skill and like 
any skill it gets better with deliberate practice 
and a library of patterns that work.

This prompt library was built over months of daily AI 
tool usage across project management, operations, 
logistics, and cybersecurity work. Every prompt here 
has been tested, refined, and proven to produce 
outputs that are actually useful rather than outputs 
that need to be completely rewritten before they 
are usable.

The prompts are organized by use case so you can 
find what you need quickly. Each one includes the 
recommended tool, the prompt template with variables 
you replace with your own context, and notes on 
what makes it work.

**The Golden Rule of Prompting**

The more context you give, the better the output. 
Tell the AI who you are, what you are trying to 
accomplish, who the audience is, what format you 
need, and what constraints you are working within. 
A prompt that takes 30 seconds longer to write 
saves 10 minutes of editing on the back end.

## SECTION 1: Project Management Prompts

---

### PM-001: Project Charter Draft

**Best Tool:** ChatGPT  
**Use Case:** Starting a new project and need a structured charter quickly

**Prompt:**
```
You are an experienced PMP-certified project manager. 
I need you to write a professional project charter 
for the following project:

Project Name: [name]
Business Problem or Opportunity: [describe the problem 
this project solves or the opportunity it captures]
Proposed Solution: [what the project will deliver]
Key Stakeholders: [list main stakeholders and their roles]
Estimated Timeline: [rough timeline]
Estimated Budget: [budget range]
Department: [department owning the project]
Methodology: [Agile / Waterfall / Hybrid]

Format the charter with these sections: Project Purpose, 
Objectives, Scope In and Out, Deliverables, Stakeholders, 
Timeline Overview, Budget Overview, Assumptions, 
Constraints, and Risks. Write in a professional but 
clear tone; avoid jargon where plain language works better.
```

**Why It Works:** Giving the AI the methodology and 
stakeholder context upfront produces a charter that 
actually reflects the project rather than a generic template.

### PM-002: Risk Register Population

**Best Tool:** ChatGPT  
**Use Case:** Populating an initial risk register for a new project

**Prompt:**
```
You are a PMP-certified project manager with expertise 
in risk management. I am managing the following project:

Project Type: [type of project]
Industry: [your industry]
Team Size: [number of people]
Timeline: [duration]
Key Deliverables: [main things being delivered]
Known Constraints: [budget, resources, technology, etc.]

Identify the top 10 risks for this project. For each risk 
provide: risk description, category (schedule, budget, 
resource, technical, security, external), likelihood 
rating (1 to 5), impact rating (1 to 5), risk score, 
and an initial mitigation strategy. Format as a table.
```

**Why It Works:** Providing project specifics produces 
risks that are actually relevant rather than a generic 
list that applies to every project equally.

### PM-003: Meeting Agenda Generator

**Best Tool:** ChatGPT  
**Use Case:** Preparing a focused, efficient meeting agenda

**Prompt:**
```
Create a professional meeting agenda for the following:

Meeting Type: [kickoff / status update / retrospective /

Attendees: [list roles, not names]
```

## SECTION 2: Operations and Logistics Prompts

---

### OPS-001: SOP Writer

**Best Tool:** ChatGPT  
**Use Case:** Drafting a new Standard Operating Procedure

**Prompt:**
```
You are an operations manager with PMP certification 
writing a Standard Operating Procedure for a professional 
portfolio. Write a detailed, human-sounding SOP for 
the following process:

Process Name: [name]
Department: [department]
Purpose: [why this process exists and what problem it solves]
Who It Applies To: [roles covered by this SOP]
Tools Used: [list relevant tools]
Key Steps: [list the main steps in the process]
Security Considerations: [any security or compliance factors]
KPIs: [how success is measured]

Write in a professional but conversational tone. 
Vary the format throughout; do not use the same table 
or list format for every section. Make it sound like 
a real operations professional wrote it from experience, 
not like a template was filled in. Include a revision 
history table at the end.
```

---

### OPS-002: Vendor Evaluation Summary

**Best Tool:** Perplexity  
**Use Case:** Researching a vendor before onboarding or contract renewal

**Prompt:**
```
Research the following vendor and provide a comprehensive 
evaluation summary:

Vendor Name: [name]
Service They Provide: [description]
Industry: [your industry]

Cover the following in your research:
1. Company background and financial stability
2. Client reputation and any public reviews or case studies
3. Any known security incidents or data breaches in 
   the past 3 years
4. Industry certifications held (SOC 2, ISO 27001, etc.)
5. Any legal disputes or regulatory actions
6. Competitive alternatives and how this vendor compares

Provide sources for all information. Flag anything 
that would be a concern from a security or operational 
risk perspective.
```

**Why Perplexity:** Perplexity searches the live web 
and cites sources; critical for vendor research where 
current and verifiable information matters more than 
AI-generated content.

## SECTION 3: Cybersecurity and Risk Prompts

---

### SEC-001: Risk Assessment Summary

**Best Tool:** ChatGPT  
**Use Case:** Drafting a risk assessment summary for leadership

**Prompt:**
```
You are a cybersecurity professional with a Master's 
degree in Cybersecurity Management and Policy. Write 
a risk assessment summary for the following scenario:

Organization Type: [type of organization]
Assessment Scope: [systems, processes, or areas assessed]
Top Risks Identified: [list 5 to 7 risks with brief descriptions]
Current Controls in Place: [list existing controls]
Gaps Identified: [where controls are missing or insufficient]
Regulatory Requirements: [applicable frameworks or regulations]

Write the summary in two versions:
1. Technical version for IT Security team
2. Executive version for non-technical leadership

Both versions should cover the same risks but the 
executive version uses plain language, business impact 
framing, and avoids technical jargon entirely.
```

### SEC-002: Security Policy Draft

**Best Tool:** ChatGPT  
**Use Case:** Drafting a new security policy document

**Prompt:**
```
Write a professional information security policy 
for the following:

Policy Name: [name]
Organization Type: [type]
Policy Purpose: [what behavior or control this policy governs]
Scope: [who and what it applies to]
Key Requirements: [what the policy requires people to do or not do]
Consequences of Non-Compliance: [what happens if violated]
Applicable Frameworks: [NIST, ISO 27001, HIPAA, GDPR, etc.]

Write in clear, accessible language that a non-technical 
employee can understand and follow. Avoid legal jargon 
where plain language works. Include purpose, scope, 
policy statements, roles and responsibilities, 
enforcement, and revision history sections.
```

**Best Tool:** ChatGPT  
**Use Case:** Documenting a security incident after resolution

**Prompt:**
```
Write a formal security incident report based on 
the following information:

Incident ID: [ID]
Date and Time Detected: [date and time]
```

## SECTION 4: Communication and Stakeholder Prompts

---

### COM-001: Difficult Email Writer

**Best Tool:** ChatGPT  
**Use Case:** Writing a professional email for a sensitive or difficult situation

**Prompt:**
```
Write a professional email for the following situation:

Situation Type: [delivering bad news / escalating an issue / 
pushing back on a request / declining a vendor / 
addressing a missed deadline / requesting more resources]
Sender Role: [your role]
Recipient Role: [their role and relationship to you]
Key Message: [what you need to communicate]
Desired Outcome: [what you want to happen after they read it]
Tone Required: [firm but respectful / empathetic / direct / 
collaborative]
Any Context: [relevant background the reader needs]

Write three versions of this email representing 
different approaches to achieving the desired outcome. 
Label each version with the strategic approach it uses. 
Keep each version under 200 words.
```

### COM-002: Executive Presentation Outline

**Best Tool:** Gamma  
**Use Case:** Building a presentation for leadership or stakeholders

**Prompt for Gamma:**
```
Create a professional presentation on the following topic:

Topic: [topic]
Audience: [who will see this presentation]
Purpose: [inform / persuade / request approval / report results]
Key Points to Cover: [list 5 to 7 main points]
Data or Metrics to Include: [any numbers or results]
Desired Action from Audience: [what you want them to do after]
Tone: [formal / professional / conversational]
Slide Count: [approximate number of slides]

Include an executive summary slide, one slide per 
key point, a data visualization slide where relevant, 
and a clear call to action on the final slide.
```

**Why Gamma:** Gamma produces visually polished 
presentations directly from prompts in minutes. 
Use it for the initial structure then refine in 
PowerPoint for final delivery.

**Best Tool:** ChatGPT  
**Use Case:** Preparing talking points for a verbal stakeholder update

**Prompt:**
```
I need to give a verbal project update to [audience: 
executives / clients / team members] in [time available] 
minutes. Prepare a structured talking points script 
based on the following:

What Has Been Accomplished: [key completions]

attention and a clear closing that specifies the ask.
```

## SECTION 5: Research and Analysis Prompts

---

### RES-001: Competitive Research

**Best Tool:** Perplexity  
**Use Case:** Researching industry trends, competitors, or market intelligence

**Prompt:**
```
Research the following topic and provide a comprehensive 
current intelligence summary:

Topic: [specific research topic]
Industry Context: [your industry]
Time Frame: [focus on information from the past 12 months]
Specific Questions to Answer:
1. [Question 1]
2. [Question 2]
3. [Question 3]

Provide cited sources for all factual claims. 
Highlight any findings that represent a significant 
change from previous industry norms. Flag information 
that is conflicting across sources.
```

**Why Perplexity:** For research requiring current, 
cited information Perplexity consistently outperforms 
ChatGPT because it searches the live web and shows 
its sources; critical for business intelligence work.

### RES-002: Document Summarizer

**Best Tool:** ChatGPT  
**Use Case:** Extracting key information from long documents quickly

**Prompt:**
```

contract / research paper / meeting transcript] below.
```

### DOC-001: Training Material Creator

**Best Tool:** ChatGPT  
**Use Case:** Building training content for a new process or tool

**Prompt:**
```
Create a training guide for the following:

Topic: [what is being taught]
Audience: [who will be trained; their existing knowledge level]
Learning Objectives: [what they should be able to do after training]
Format: [written guide / slide deck outline / quick reference card]
Length: [approximate length or number of pages]
Tools or Systems Covered: [list relevant tools]

Include an introduction explaining why this topic matters, 
clearly numbered steps for any procedures, a FAQ section 
addressing the most common questions or mistakes, 
a quick reference summary at the end, and a knowledge 
check with 5 multiple choice questions to test comprehension.

Write in plain language accessible to someone with 
no prior knowledge of the topic.
```

### DOC-002: Job Description Writer

**Best Tool:** ChatGPT  
**Use Case:** Writing or updating a job description for an operations role

**Prompt:**
```
Write a compelling and accurate job description for 
the following role:

Job Title: [title]
Department: [department]
Reports To: [manager title]
Employment Type: [full-time / part-time / contract]
Location: [on-site / remote / hybrid]
Core Responsibilities: [list 6 to 8 main duties]
Required Qualifications: [must-have education, experience, certs]
Preferred Qualifications: [nice-to-have skills]
```

## SECTION 7: Automation and Process Design Prompts

---

### AUT-001: Automation Opportunity Identifier

**Best Tool:** ChatGPT  
**Use Case:** Finding automation opportunities in an existing workflow

**Prompt:**
```
I am going to describe a current manual workflow. 
Analyze it and identify every step that could 
realistically be automated using tools like Zapier, 
Make.com, or AI tools available in 2025.

Workflow Description: [describe the current process 
step by step as it actually happens today]

For each automation opportunity identified provide:
1. Which step can be automated
2. Recommended tool or platform
3. Estimated time saved per occurrence
4. Complexity to implement (Low / Medium / High)
5. Any risks or considerations

Prioritize recommendations by impact; highest time 
savings and lowest implementation complexity first.
```

---

### AUT-002: Zapier Workflow Planner

**Best Tool:** ChatGPT  
**Use Case:** Planning a Zapier automation before building it

**Prompt:**
```
Help me plan a Zapier automation workflow for 
the following use case:

What I Want to Automate: [describe the outcome you want]
Trigger Event: [what action should start the automation]
Apps Involved: [list all apps that need to be connected]
Data to Pass Between Apps: [what information moves 
from one app to another]
Actions Required: [what should happen at each step]
Any Conditions or Filters: [steps that should only 
happen under certain conditions]
Error Handling: [what should happen if a step fails]

Provide a step by step workflow plan I can follow 
to build this in Zapier, including recommended 
Zapier apps, trigger and action configurations, 
and any filter or path steps needed. Flag any 
steps that may require a paid Zapier plan.
```

### AUT-003: Cursor Coding Assistant

**Best Tool:** Cursor  
**Use Case:** Building or debugging a simple script or automation

**Prompt:**
```
I need help building the following:

What I Want to Build: [describe what the code should do]
My Technical Level: [beginner / intermediate / advanced]
Language or Platform: [Python / JavaScript / Google Apps 
Script / other]
Input Data: [what data the code will work with]
Output Required: [what the code should produce]
Any Constraints: [things the code must or must not do]

Please:
1. Explain what approach you are taking and why
2. Write the complete code with comments explaining 
   each section
3. Show me how to test it with a simple example
4. Tell me what could go wrong and how to handle it

Write for my technical level; explain concepts I 
may not know rather than assuming I will figure it out.
```

**Why Cursor:** For actual code writing and debugging 
Cursor is purpose-built for development work. It 
understands code context across files and produces 
more reliable, production-ready output than 
general-purpose AI tools for coding tasks.

## 🔐 Responsible AI Use Reminders

Before sending any prompt to an AI tool, check 
the following:

Does this prompt contain real employee names or PII? 
If yes, replace with generic placeholders before submitting.

Does this prompt contain confidential financial data? 
If yes, use approximate figures or ranges rather 
than exact numbers.

Does this prompt contain sensitive security information 
such as system names, vulnerability details, or 
incident specifics? If yes, use a self-hosted or 
enterprise version of the tool only.

Does this prompt contain client or customer information? 
If yes, check your organization's AI acceptable use 
policy before proceeding.

AI tools are powerful assistants; they are not a 
replacement for professional judgment, institutional 
knowledge, or accountability. Every output from 
these tools should be reviewed, refined, and owned 
by the person submitting it before it goes anywhere.
