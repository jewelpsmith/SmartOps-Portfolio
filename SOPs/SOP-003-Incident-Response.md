## 1. Purpose

This SOP defines the structured process for identifying, containing, 
eradicating, and recovering from security incidents. It ensures a 
swift, coordinated response that minimizes operational disruption 
and protects organizational assets.

---

## 2. Scope

Applies to all security incidents affecting systems, networks, data, 
and personnel across Operations and Logistics departments.

---

## 3. Roles & Responsibilities

| Role | Responsibility |
|---|---|
| Incident Response Lead | Coordinate and oversee response |
| IT Security Analyst | Investigate and contain incident |
| Project Manager | Track response timeline and tasks |
| Operations Manager | Communicate with leadership |
| Legal/Compliance | Assess regulatory reporting requirements |

---

## 4. Incident Severity Levels

| Level | Severity | Description | Response Time |
|---|---|---|---|
| P1 | Critical | Full system compromise / data breach | Immediate |
| P2 | High | Partial system compromise / malware | Within 1 hour |
| P3 | Medium | Suspicious activity / policy violation | Within 4 hours |
| P4 | Low | Minor anomaly / failed login attempts | Within 24 hours |

---

## 5. Incident Response Procedure (NIST Framework)

### Phase 1: Preparation
- [ ] Maintain up-to-date incident response plan
- [ ] Ensure all staff complete security awareness training
- [ ] Confirm all systems have logging and monitoring enabled
- [ ] Maintain emergency contact list for response team
- [ ] Test incident response plan quarterly

### Phase 2: Identification
- [ ] Detect incident via monitoring tools or user report
- [ ] Document initial findings (time, affected systems, symptoms)
- [ ] Assign severity level (P1-P4)
- [ ] Notify Incident Response Lead immediately
- [ ] Open incident ticket in tracking system

### Phase 3: Containment
- [ ] Isolate affected systems from network
- [ ] Preserve evidence (logs, screenshots, affected files)
- [ ] Block malicious IPs or accounts
- [ ] Implement short-term fixes to prevent spread
- [ ] Notify Operations Manager and Legal/Compliance

### Phase 4: Eradication
- [ ] Identify root cause of incident
- [ ] Remove malware or unauthorized access
- [ ] Patch vulnerabilities exploited
- [ ] Verify systems are clean before restoration
- [ ] Document all actions taken

### Phase 5: Recovery
- [ ] Restore systems from clean backups
- [ ] Monitor systems closely post-recovery
- [ ] Confirm normal operations resumed
- [ ] Notify stakeholders of resolution
- [ ] Update system access controls as needed

### Phase 6: Lessons Learned
- [ ] Conduct post-incident review within 72 hours
- [ ] Document what worked and what didn't
- [ ] Update incident response plan accordingly
- [ ] Provide additional training if needed
- [ ] Submit final incident report to leadership
