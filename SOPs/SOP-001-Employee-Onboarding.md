## 1. Purpose

This Standard Operating Procedure defines the structured process 
for onboarding new employees in a secure, efficient, and compliant 
manner. It ensures all personnel are properly credentialed, trained, 
and integrated into operations while maintaining organizational 
security posture.

---

## 2. Scope

Applies to all full-time, part-time, and contract employees across 
Operations and Logistics departments.

---

## 3. Roles & Responsibilities

| Role | Responsibility |
|---|---|
| HR Manager | Initiate onboarding, collect documentation |
| IT Security | Provision accounts, enforce access controls |
| Department Manager | Assign tasks, introduce team |
| Project Manager | Ensure onboarding timeline is met |
| New Employee | Complete training, sign policies |

---

## 4. Security Considerations (CMP Framework)

- All new hires must complete **Security Awareness Training** within 72 hours
- Access is granted on a **least privilege** basis
- All credentials provisioned through **Identity & Access Management (IAM)**
- New employees must sign **Acceptable Use Policy (AUP)** before system access
- Background verification required before access to sensitive systems

---

## 5. Onboarding Procedure

### Phase 1: Pre-Arrival (1 Week Before Start Date)
- [ ] Send offer letter and required documents checklist
- [ ] Initiate background check
- [ ] Submit IT access request form
- [ ] Prepare workstation and equipment
- [ ] Schedule orientation and training sessions

### Phase 2: Day 1 — Orientation
- [ ] Welcome meeting with manager
- [ ] ID badge and physical access issued
- [ ] System credentials provisioned (email, Jira, Notion, etc.)
- [ ] Sign Acceptable Use Policy & confidentiality agreement
- [ ] Complete Security Awareness Training (mandatory)
- [ ] Office/facility tour

### Phase 3: Week 1 — Integration
- [ ] Introduction to team and key stakeholders
- [ ] Review of SOPs relevant to role
- [ ] Shadow existing team members
- [ ] Set up project management tools (Jira, Trello, Notion)
- [ ] Complete role-specific training modules

### Phase 4: Days 30/60/90 — Check-ins
- [ ] 30-day performance check-in with manager
- [ ] 60-day access review — confirm permissions are appropriate
- [ ] 90-day full performance review and feedback session
